## Features shipped (v0.9.2-alpha)

### Feature A — Authentik Reputation Policy
- `_authentik_setup_reputation_policy()` — runs automatically on first startup via the post-update migration ladder, binds a reputation policy to `ldap-authentication-flow`
- Every failed login decrements the source-IP score; score below threshold (default −5) blocks the login before the password stage runs
- UI card on the Authentik page: toggle, threshold config, live top-IPs table, "Clear All Scores"
- API: `GET /api/authentik/reputation/status`, `POST /api/authentik/reputation/config`, `POST /api/authentik/reputation/scores/clear`

### Feature B — SSH Fail2ban Jail
- Opt-in sshd jail using the built-in `sshd` filter; monitors `/var/log/auth.log`
- Defaults: maxretry=3, findtime=10 min, bantime=60 min
- Guard Dog email alert fires on ban (reuses `infratak-guarddog.conf`)
- UI card on the Fail2ban page: toggle, stats, config, whitelist (ignoreip), ban list, unban button
- API: `GET /api/fail2ban/ssh/status`, `POST /api/fail2ban/ssh/config`, `POST /api/fail2ban/ssh/unban`

### Feature C — TAK Server Snapshots
- `_tak_snapshot(label)` captures: CoreConfig.xml, UserAuthenticationFile.xml, takserver.default (JVM heap), certs/ directory, pg_dump of the cot database
- Pre-upgrade snapshot: taken automatically at the start of every `run_takserver_upgrade()`; upgrade aborts if the snapshot fails
- Scheduled snapshots: systemd timer (daily at 02:00 by default); operator configures schedule and retention count from the UI
- Manual snapshot: "Take Snapshot Now" button on the TAK Server page
- Off-box export: "Download" button streams a `.tar.gz` per snapshot
- API: `GET /api/takserver/snapshots`, `POST /api/takserver/snapshot/run`, `GET /api/takserver/snapshot/status`, `GET /api/takserver/snapshot/<label>/download`, `DELETE /api/takserver/snapshot/<label>`, `GET/POST /api/takserver/snapshot/schedule`

### Feature D — TAK Server Rollback
- `_tak_rollback(label)` — stops takserver, restores CoreConfig.xml + UserAuthenticationFile.xml + takserver.default + certs/, runs pg_restore --clean, starts takserver
- Snapshots taken before v0.9.2 won't have UserAuthenticationFile.xml — rollback logs a notice and skips that one step
- UI: Restore / Download / Delete buttons per snapshot in the snapshot table on the TAK Server page
- API: `POST /api/takserver/rollback`, `GET /api/takserver/rollback/log`

### Feature E — infra-TAK Console Rollback
- `update_apply()` records `settings.console_rollback = {available, version, tag, snapshot_at}` before every "Update Now"
- Console page shows a yellow "Rollback available — vX.Y.Z" bar after each update
- One click checks out the previous tag from GitHub and restarts
- Rollback availability is cleared after use (one rollback per update cycle)
- API: `POST /api/console/rollback`

### Feature F — TAK Server Plugins
- New collapsible "TAK Server Plugins" section on the TAK Server page (visible when TAK is installed)
- Plugin JARs live at `/opt/tak/lib/`; YAML configs at `/opt/tak/conf/plugins/<classname>.yaml`
- UI: installed plugin table (name, config status, Edit config, Remove); single upload area for .jar or .yaml; restart banner after any file change; inline YAML editor per plugin
- API: `GET /api/takserver/plugins/list`, `POST /api/upload/takserver-plugin`, `POST /api/takserver/plugins/install-jar`, `POST /api/takserver/plugins/install-yaml`, `GET/POST /api/takserver/plugins/config/<classname>`, `POST /api/takserver/plugins/remove/<jarname>`

## Bug fix — Guard Dog socket leak (console hang after ~3 days)
- **Symptom:** After ~3 days of uptime, port 5001 stops accepting connections; gunicorn accumulates 140+ leaked sockets
- **Root cause:** Five `urllib.request.urlopen()` calls in Guard Dog health checks never called `resp.close()`. Affected: `authentik` service check, `nodered` service check, `remotedb` monitor, `authentik_http` monitor, `nodered_http` monitor
- **Fix:** Wrapped all five with `with urllib.request.urlopen(...) as resp:` — no logic change, sockets released immediately after each check

## Security hardening (applied via `_auto_*` post-update patchers in start.sh)

### Container hardening (`_auto_authentik`, `_auto_takportal`, `_auto_cloudtak`, `_auto_nodered`)
- `cap_drop: ALL` + `no-new-privileges: true` added to: Authentik server, Authentik worker, Authentik ldap, TAK Portal, CloudTAK api/events/media, Node-RED
- Authentik worker: `/var/run/docker.sock` mount removed (was inherited from upstream quickstart; infra-TAK uses embedded outpost + standalone LDAP container so it was never needed — with it mounted a compromised worker was equivalent to host root)

### Node-RED hardening
- Image pinned to `nodered/node-red:4.0` (was `:latest`)
- `user: 1000:1000`, `mem_limit: 2g`, `restart: unless-stopped`
- `env_file: .env` for optional adminAuth
- Local deploy: scoped per-file cert mounts (`/opt/tak/certs/files/<name>:/certs/<name>:ro`) instead of mounting the whole tree
- Remote deploy: port binding corrected from `0.0.0.0:1880` → `127.0.0.1:1880`

## CVE-2026-31431 ("Copy Fail") — action required on VPS
A Linux kernel vulnerability published 2026-04-29 allows unprivileged container escape to host root. The only real fix is patching the kernel:

```bash
apt update && apt full-upgrade
reboot
```

The reboot triggers Guard Dog's boot sequencer — all containers restart in the correct order automatically.
